A peripheral that can be the destination for DMA data

By ‘destination,’ we mean that it receives data from a DMA transfer. A destination would be a peripheral that could send data out of processor memory, like a UART transmitter.

Safety

Destination should only be implemented on peripherals that are DMA capable. This trait should be implemented by HAL authors who are exposing DMA capable peripherals.

fn destination_address(&self) -> *const E[src]

Returns a pointer to the register into which the DMA channel writes data

This is the register that software writes to when sending data to a device. The type of the pointer describes the type of reads the DMA channel performs when transferring data.

fn enable_destination(&mut self)[src]

Perform any actions necessary to enable DMA transfers

Callers use this method to put the peripheral into a state where it can accept transfers from a DMA channel.

fn disable_destination(&mut self)[src]

Perform any actions necessary to disable or cancel DMA transfers

This may include undoing the actions in enable_destination.
